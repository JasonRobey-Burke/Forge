# Persona: Alex Rivera

## Demographics
- **Role/Title:** Tech Lead at a 4-person startup
- **Age Range:** 30-39
- **Tech Comfort Level:** Expert
- **Industry/Domain:** B2B SaaS Development

## Goals & Motivations
- **Primary Goal:** Create well-structured specs that AI coding agents can consume to implement features autonomously
- **Secondary Goals:** Track spec progress across the team, enforce quality gates before specs go to implementation
- **Underlying Motivation:** Wants to ship features faster with AI assistance but needs structured specs to get good results

## Context & Environment
- **Typical Workflow:** Creates 3-5 specs per week, reviews team specs daily, uses the Flow Board to manage sprint work
- **Devices Used:** Desktop (1440px+), ultrawide monitor
- **Time Constraints:** 15-30 min per spec authoring session, quick board checks throughout the day
- **Frequency of Use:** Daily, multiple times per day

## Pain Points & Frustrations
- **Current Frustrations:** Tools like Jira have too much ceremony; wants something lightweight but structured
- **Accessibility Needs:** None specific
- **Common Mistakes:** Forgets to link expectations before trying to move specs to Ready

## Success Criteria
- **Task Completion:** Spec exported as YAML/Markdown, handed to AI agent, produces working code
- **Time Expectation:** Under 20 minutes to author a complete spec
- **Satisfaction Indicators:** "The AI agent understood exactly what to build from this spec"

## Key Questions This Persona Would Ask
1. "Can I see at a glance which specs are blocked and why?"
2. "How quickly can I create a new spec from an existing template?"
3. "Can I drag specs between phases without losing my place?"
4. "What's the fastest path from product idea to AI-ready spec?"
