# Persona: Dana Kim

## Demographics
- **Role/Title:** Product Manager / part-time developer on a small team
- **Age Range:** 28-35
- **Tech Comfort Level:** Intermediate
- **Industry/Domain:** Developer Tools / Productivity

## Goals & Motivations
- **Primary Goal:** Define products with clear intentions and track spec progress across the board
- **Secondary Goals:** Configure WIP limits, understand bottlenecks, export specs for stakeholder review
- **Underlying Motivation:** Needs visibility into what the team is working on and where things are stuck

## Context & Environment
- **Typical Workflow:** Sets up products and intentions, reviews the Flow Board daily, exports specs for external review
- **Devices Used:** 13" MacBook (1440×900 effective), occasionally a tablet for quick board checks
- **Time Constraints:** 10-15 minutes for board review, 30 min for product setup
- **Frequency of Use:** Daily board checks, weekly product/intention management

## Pain Points & Frustrations
- **Current Frustrations:** Hard to get a big-picture view; wants dashboards and metrics
- **Accessibility Needs:** None specific
- **Common Mistakes:** Tries to create specs before setting up intentions/expectations (doesn't follow the hierarchy)

## Success Criteria
- **Task Completion:** Can see at a glance which specs are in which phase, identify blockers, and report to stakeholders
- **Time Expectation:** Under 5 minutes for a daily board review
- **Satisfaction Indicators:** "I can tell my team exactly where we are without digging through pages"

## Key Questions This Persona Would Ask
1. "Can I see a summary of how many specs are in each phase?"
2. "How do I know which specs are stuck or overdue?"
3. "Can I export a board view or summary for a status update?"
4. "What's the quickest way to set up a new product with all the hierarchy?"
